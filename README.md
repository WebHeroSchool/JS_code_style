### <a name='ex1'>ПРИМЕР 1 (Использование ===)</a>

+ При сравнении использовать === вместо ==

    ```js
    // is false хорошо
    [10] === 10    
    
    // is true плохо
    [10]  == 10    

    // is true плохо
    '10' == 10 

    // is false хорошо    
    '10' === 10    

    // is true плохо
    []   == 0

    // is false хорошо     
    [] ===  0     

    // is true but true == "a" is false  плохо
    '' == false

    // is false  хорошо     
    '' ===   false   
    ```

### <a name='ex2'>ПРИМЕР 2 (Уменьшить доступ к переменным и свойствам)</a>

+ Мы должны сократить время доступа к переменным и свойствам объектов в наших приложениях. Это необходимо потому, что каждый раз, когда мы делаем это, ЦПУ приходится снова и снова обращаться к элементу в памяти, чтобы вычислить его результаты. Поэтому мы должны делать это как можно меньше.

    ```js
    // плохо
    for (let i = 0; i < arr.length; i++) {
      // ...код...
    }

    // хорошо
    let length = arr.length;
    for (let i = 0; i < length; i++) {
      // ...код...
    }
    ```

### <a name='ex4'>ПРИМЕР 4 (Избегайте использование глобальных переменных)</a>

+ Старайтесь сводить к минимуму использование глобальных переменных.
Сюда относятся все типы данных, объекты и функции. 
Дело в том, что глобальные переменные и функции могут переписываться
другими скриптами. Вместо этого используйте локальные переменные и учитесь
использовать замыкание.

    ```js
   // плохо
    let a = 1;
    function fun() {
      return a
    }

   // хорошо
    function fun() {
      let a = 1;
    function fun1() {
       let b = 5;
       }
    }
    ```

### <a name='ex5'>ПРИМЕР 5 (Старайтесь не использовать new Object())</a>

+ Используйте `{}` вместо `new Object()`.
+ Используйте `''` вместо `new String()`.
+ Используйте `0` вместо `new Number()`.
+ Используйте `false` вместо `new Boolean()`.
+ Используйте `[]` вместо `new Array()`.
+ Используйте `/()/` вместо `new RegExp()`.
+ Используйте `function() {}` вместо `new Function()`.

    ```js
    // плохо
    var x1 = new Object();
    var x2 = new String();
    var x3 = new Number();
    var x4 = new Boolean();
    var x5 = new Array();    
    var x7 = new Function();

    // хорошо
    var x1 = {}; // new object
    var x2 = ''; // new primitive string
    var x3 = 0; // new primitive number
    var x4 = false; // new primitive boolean
    var x5 = []; // new array object
    var x6 = /()/; // new regexp object
    var x7 = function() {}; // new function object
    ```

### <a name='ex6'>ПРИМЕР 6 (Точка с запятой)</a>

+ Используйте точечную нотацию для доступа к свойствам и методам.

    ```js
    // плохо
    let luke = {}
    let leia = {}
    [luke, leia].forEach(jedi => jedi.father = 'vader')

    // хорошо
    let luke = {};
    let leia = {};
    [luke, leia].forEach((jedi) => {
      jedi.father = 'vader';
    });  
    ```
**[[⬆]](#СОДЕРЖАНИЕ)**

### <a name='ex7'>ПРИМЕР 7 (Доступ к DOM)</a>

+  Доступ к HTML DOM происходит очень медленно по сравнению с другими 
операторами JavaScript. Если вы ожидаете получить доступ к элементу DOM несколько раз, 
получите доступ к нему один раз и используйте его как локальную переменную.

    ```js
    // плохо
    document.getElementById("demo");
    document.getElementById("demo").innerHTML = "Hello";

    // хорошо
    var obj = document.getElementById("demo");
    obj.innerHTML = "Hello";
    ```

### <a name='ex8'>ПРИМЕР 8 (Комментарии)</a>

+  Используйте // для комментариев в одну строку. Размещайте комментарии на новой строке над темой комментария. Добавляйте пустую строку над комментарием.

    ```js
    // плохо
    var active = true;  // устанавливаем активным элементом

    // хорошо
    // устанавливаем активным элементом
    var active = true;

    // плохо
    function getType() {
      console.log('проверяем тип...');
      // задаем тип по умолчанию 'no type'
      var type = this._type || 'no type';

      return type;
    }

    // хорошо
    function getType() {
      console.log('проверяем тип...');

      // задаем тип по умолчанию 'no type'
      var type = this._type || 'no type';

      return type;
    }
    ```

### <a name='ex10'>ПРИМЕР 10 (Размещать скрипты в конце страницы)</a>

+  script размещаем внутри тега body в конце. Это делает загрузку страницы максимально быстрой для пользователя.

    ```js
    // плохо
    <body>
      <script src="script.js"></script>
        <!-- код страницы -->
    </body>

    // хорошо
    <body>
      <!-- код страницы -->
      <script src="script.js"></script>
    </body>
    ```
